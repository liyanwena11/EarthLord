## App Description
EarthLord is a location-based survival game set in a post-apocalyptic world of 2051. Players claim territories by walking in the real world, explore resources at real POIs (Points of Interest), build structures, and interact with other players through trading and communication systems.

**Key Innovation:** Walk to Earn gameplay that encourages outdoor exercise while delivering an immersive location-based gaming experience.

**Target Audience:** Players aged 18-35 who enjoy exploration, survival games, and gamified fitness.

**Problems Solved:**
- Lack of exercise in traditional games - GPS territory claiming encourages outdoor walking
- Lack of innovative gameplay - Real-world location-based gaming experience
- Limited social interaction - Distance-based social features (within 5km)

## Step-by-Step Testing Instructions

### Step 1: Login (2 minutes)
1. Launch the app
2. Wait for splash animation
3. Tap "Login" button
4. Enter sandbox test account credentials
5. Tap "Login"

**Expected:** Successfully logged in, see 5 tabs (Map, Territory, Resources, Communication, Profile)

### Step 2: GPS Territory Claiming (5 minutes)
**Method A: Real GPS (Recommended)**
1. Allow GPS location access
2. Go to "Map" tab
3. Tap "Start Claiming" button
4. Walk in a small circle (50-100m)
5. Tap "Stop Claiming" button
6. Wait for path closure detection

**Method B: Simulated GPS (Xcode)**
1. In Xcode: Edit Scheme → Run → Options → Add GPX file
2. Use "SimulatedRun.gpx" from project
3. Run app and simulate GPS movement

**Expected:** Territory created successfully, showing area and location

### Step 3: POI Exploration (3 minutes)
